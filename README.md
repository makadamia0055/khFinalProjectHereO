## 3. 설계의 주안점

---

1. 종합 아마추어 야구 플랫폼으로서의 기능 추구
    1. 팀 등록, 구장 예약, 경기 기록, 커뮤니티 기능을 통해 아마추어 야구 경기를 시작부터 끝까지 지원하는 종합 아마추어 야구 플랫폼 사이트의 기능
2. 야구 경기 데이터 관리
    1. 하나의 경기를 기록하기 위해서 야구 경기를 세부 단위로 나누어 기록할 수 있게 구성 
    2. 야구 경기 기록 데이터를 경기-이닝-타석 단위로 구분
        1. 경기 기록 : 한 경기의 기록
        2. 이닝 : 경기가 진행된 이닝(1~9, 선후공)
        3. 타석 : 이닝 동안 진행된 타석의 이벤트
    3. 각 단위로 구분하여, 경기 기록을 보다 정합하게 기록 할 수 있도록 설계
3. 경기 예약 → 경기 수행 → 기록 도출로 이어지는 통합적 관리
    1. 경기를 예약한 시점부터 실제 경기가 끝나고 경기의 기록이 산출될 때까지 통합적 관리 제공

## 4. 요구 사항 개발 및 관리

---

- 각 페이지 요구사항을 미리 작성하며, 작업 진행의 지침으로 삼고 개발하고자 함.
    - 팀원 별로 각 담당 부분에 대한 요구사항을 작성하고, 다른 팀원이 피드백 하는 형식으로 수행
- 그러나 상기와 같은 프로세스를 팀원 회의를 통해 정했음에도, 담당 영역에 대한 요구사항을 제대로 작성하지 않은 팀원들이 있었고, 피드백 과정도 원활하게 이루어지지 않았다.
- 또 개발 과정에서 참고하기는 했으나, 변경된 사항을 지속적으로 반영하지 않은 부분이 있었다.

## 7. 팀 내 역할

---

- 팀 내 포지션 : 팀장
    - 팀장으로 전체적 일정 관리, 진행 계획, 전체 코드 관리 등을 수행
    - 의사 결정 과정에 적극적으로 참여하고 해당 결정 과정과 내역들을 기록
    - 프로젝트 마감 전 결원과 팀원의 역량 분제로 구현하지 못한 부분을 대신 작업
- 담당 페이지
    - 팀 페이지 영역 전체 구현(Front/Back)
        - 팀 페이지를 전담하여 구현.
    - 경기 기록 페이지 및 경기 예약 페이지 프론트 및 백앤드 구현
        - 프로젝트 진행 중 여러 인적 이슈들이 발생하여, 해당 영역들을 대신 구현하였습니다.
        - 타 팀원의 작업 내용을 이어받아, 기능 구현에 필요한 프론트 요소와 백앤드 전체 작업
- DB 관리 및 설계
    - 팀원 별로 각 담당 파트에 대한 데이터베이스 도메인을 설계하고, 해당 내용을 최종적으로 팀장인 제가 확인하며 수정 및 반영하였습니다.
    - 팀 페이지에 관련된 전체 도메인 구성과 야구 경기 기록을 위한 도메인 설계를 전담
        - 팀 - 선수 - 회원의 관계와 야구 경기를 경기-이닝-타석으로 나누고, 경기 계획과 실제 경기 수행 기록을 나누어 관리하는 구조를 제시했습니다.
    - MySQL 데이터 관리 및 DB 데이터 관리
        - 트리거, 프로시저 관리

## 11. 프로젝트 참여 소감

---

첫 프로젝트를 진행하며 느낀 감정 중 가장 큰 부분은 아쉬움과 난감함이었습니다. 교육 과정에서 성적이 좋았다는 이유로 팀장을 맡게 되어, 갑작스럽게 인적 자원과 프로젝트 전체 과정을 조율하게 된 입장에 서니 난감했습니다. 학부 과정이나 대학원에서도 팀으로 활동해 본 적은 있었지만, 프로젝트처럼 장기간의 계획을 끌어가 본 적은 없었습니다.

 뭘 어떻게 해야 할 지 몰라 처음에는 팀의 워크 프로세스를 만드는데 집중했었습니다.  프로젝트 당시 정보처리기사 공부를 하고 있었으므로, 정보처리기사 과정을 공부하며 배운 여러 프로세스들을 응용하여, 세부적으로 일을 할 때 지켜야 할 컨벤션들을 만들고, 작업을 할 때 작업 내용이 담긴 일지를 작성하고, 코드 리뷰 페이지를 만드는 등의 프로세스들을 만들었습니다. 나름 효율적일 것이라고 생각해서 만든 것이지만 결국 저만 사용하거나 저조차 사용하지 않게 되었습니다. 팀원 간의 소통으로 할 수 있는 일들을 프로세스로 넘겨버린 뒤라, 서로 간의 소통도 소원하게 된 점도 있었습니다. 이런 결과들로 제 접근 방법의 미흡했던 부분을 절감했습니다.

또 기획 자체를 너무 방대하게 잡았던 점도 아쉽습니다. 기획 자체는 아마추어 야구를 하고 있던 팀원이 제시한 아이템이 좋아 보여 선정했지만, 야구라는 도메인 영역 범위를 (나름 대비한다고 했지만) 과소평가한 점이 큰 패착이었던 것 같습니다. 특히 해당 부분들을 설계 및 관리 할 수 있는 역량을 가진 팀원이 저 밖에 없었던 터라, 저 스스로는 야구의 스키마를 설계하면서 재미도 느꼈지만, 해당 파트에 집중한다고 다른 팀원 영역을 도와주지 못한 점이 아쉽습니다. 

코드 자체를 대하고, DB의 도메인을 설계하는 부분에서 재미를 느낄 수 있다는 것을 알게 된 점은 좋았습니다. 언어와 프레임워크를 배우고, 해당 영역이 돌아가는 메커니즘을 이해하려고 시도하면서 코딩이라는 영역에 대한 흥미와 적성을 확인했습니다. 물론 저 자신의 모자란 부분도 확실하게 알 수 있었습니다. 프로젝트를 진행하며 웹 개발의 영역을 훑어보며, 전체 영역을 다 파악하지는 못했지만, 개략적으로 웹 개발에 대한 단초를 얻은 점도 좋았습니다. 

낮에는 공부를 하고, 오후에는 코드를 작성하는 일상을 반복하는 것도 의미가 있었습니다. 대학원 수료 후 한동안 삶에서 목표를 잃고 방황하며 자신을 잃어버린 것 같았습니다. 스스로를 한 없이 흩어져서 잡을 수 없던 상황이었습니다. 이런 시점에서 다시 자리에 앉아 공부를 하고 코딩을 하는 것은 힘들었습니다. 그러나 다시금 눈 앞에 목표를 정하고 꾸준히 작업하는 과정을 통해 스스로를 단조하려 하였고, 그 과정에서 여러 피드백을 받으며 자신을 다잡게 되었습니다. 

갈무리하는 시점에서 평가해보면 그리 성공적이지는 못한 프로젝트였습니다. 미흡한 소통과 기획의 실패로 난관을 겪었습니다. 그러나 그 실패에서 확실하게 배운 점은 있었습니다. 또 코딩이라는 영역에서 저 자신의 강점과 미흡한 부분을 알아갈 수 있었습니다. 그리고 매일 자리에 앉아 반복하는 과정에서 의미를 찾았습니다. 부족하고 부끄러운 부분은 많습니다. 하지만 길을 걷다 문득 돌아보면 항상 부끄러운 부분이 시작점이 아닌가 합니다. 개발자로서 길을 걷다 돌아보며 부끄러워할 미래의 제 모습을 그려보면서 이번 프로젝트에 대한 소감을 정리하겠습니다.
